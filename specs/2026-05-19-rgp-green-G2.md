## Findings Fixed

### B6-1 — Azure raw HTTP error body surfaced into `LlmError` / tracing / CLI

**Root cause files and lines before fix:**

- `crates/clx-core/src/llm/azure.rs:196-211` — `map_response` built
  `format!("{body} (x-request-id: {request_id})")` from the unbounded raw
  HTTP response body and embedded it verbatim into `LlmError::Auth`,
  `LlmError::DeploymentNotFound`, `LlmError::ContentFilter`,
  `LlmError::Server`.
- `crates/clx-core/src/policy/llm.rs:162` — `warn!("L1 LLM unavailable: {}",
  e)` logged the `LlmError` Display with no `redact_secrets` call.
- `crates/clx/src/commands/health.rs:91/102/694` — the Azure `endpoint` was
  printed verbatim; `Err(e).to_string()` was stored and printed without
  redaction.

**Fix applied:**

1. **`crates/clx-core/src/llm/azure.rs`** — added `build_error_summary(status,
   body, request_id)` helper:
   - Calls `redact_secrets(body.trim())` on the **full** body first (so host
     patterns spanning a future truncation boundary are always caught).
   - Truncates the already-redacted string to `MAX_BODY_EXCERPT_BYTES = 80`
     bytes via `truncate_utf8` (UTF-8-safe, walks backwards from boundary).
   - Redacts `x-request-id` through `redact_secrets` as a precaution.
   - Formats as `status={code} body={excerpt:?} x-request-id={rid}`.
   - Replaced all four error arms in `map_response` to use this helper
     instead of `body_with_id`.

2. **`crates/clx-core/src/policy/llm.rs:162`** — changed to:
   ```rust
   warn!("L1 LLM unavailable: {}", redact_secrets(&e.to_string()));
   ```
   Added `use crate::redaction::redact_secrets` import.

3. **`crates/clx/src/commands/health.rs`** — two sink points patched:
   - Azure endpoint: `redact_secrets(&a.endpoint)` instead of
     `a.endpoint.clone()`.
   - `create_llm_client_by_name` error: `redact_secrets(&e.to_string())`
     instead of `e.to_string()`.
   Added `use clx_core::redaction::redact_secrets` import.

**Before:** `LlmError::Auth` Display: `authentication failed: {"error":{"code":"401","message":"Access denied. Resource: https://REAL-TENANT.openai.azure.com/openai/deployments/..."}}`

**After:** `LlmError::Auth` Display: `authentication failed: status=401 body="{\"error\":{\"code\":\"401\",\"message\":\"Access denied. Resource: https://***AZURE-HOST" x-request-id=rid-0001`

The tenant hostname is gone; status code and correlation ID are preserved for debuggability.

### B6-2 — `redact_secrets` has no Azure/OpenAI tenant host pattern

**Root cause file before fix:**

- `crates/clx-core/src/redaction.rs:20-272` — patterns covered API key
  prefixes, keyword=value pairs, Bearer/Basic tokens, export patterns. No
  hostname/URL rule existed.

**Fix applied:**

`crates/clx-core/src/redaction.rs` — added:

1. **`AZURE_HOST_SUFFIXES`** constant:
   ```rust
   const AZURE_HOST_SUFFIXES: &[&str] = &[
       ".openai.azure.com",
       ".azure-api.net",
       ".cognitiveservices.azure.com",
   ];
   ```

2. **`AZURE_HOST_REDACTED`** replacement token: `"***AZURE-HOST-REDACTED***"`.

3. **`redact_azure_hosts(text: &str) -> String`** private helper — two passes:
   - Pass 1: scans for `https://` / `http://` (case-insensitive via a
     pre-computed lowercase copy), extracts the authority component, and if
     it ends with any Azure suffix replaces the authority with
     `***AZURE-HOST-REDACTED***`. Scheme and path are kept for context.
   - Pass 2: tokenises the pass-1 output on whitespace/punctuation and
     replaces any bare hostname token whose authority portion ends with an
     Azure suffix. Tokens already containing the redaction marker are skipped.

4. **`redact_secrets` entry point** — before the existing passes, applies
   `redact_azure_hosts` when the text contains `://` or any Azure suffix
   (fast-path exit for unrelated text). The result is then passed through
   the existing keyword/prefix/bearer passes as before.

**Host patterns added (3):** `*.openai.azure.com`, `*.azure-api.net`,
`*.cognitiveservices.azure.com`.

**Non-regression contract:** URLs not ending with these suffixes (e.g.
`https://docs.example.com/`, `http://127.0.0.1:11434/`) are left verbatim.
The `synthetic-tenant.example-openai.invalid` host used in the RED R2 PoC is
intentionally NOT scrubbed (the `.invalid` TLD is not in the suffix list —
this is the correct contract; the fix targets real Azure suffixes only).

**Before:** `redact_secrets(SYNTH_AZURE_BODY)` — `synthetic-tenant.openai.azure.com` survives.

**After:** `redact_secrets(body_with_real_azure_suffix)` — host replaced with `***AZURE-HOST-REDACTED***`.

## Bounded-Error + Redaction Design

The design follows a defence-in-depth layering:

```
Azure HTTP response
  └─ map_response (azure.rs)
       └─ build_error_summary(status, body, request_id)
            ├─ redact_secrets(body.trim())   ← full-body redaction first
            │    └─ redact_azure_hosts()     ← host patterns (B6-2)
            │    └─ prefix/keyword/bearer passes
            └─ truncate_utf8(redacted, 80)  ← size cap after redaction
            └─ format! → LlmError variant   ← bounded, structured, redacted
                 └─ Display → tracing sink
                      └─ warn!("L1 LLM unavailable: {}", redact_secrets(&e.to_string()))
                                                          ↑ second-chance scrub (B6-1)
```

The double-redaction (in `build_error_summary` and again at the `warn!` sink)
is intentional: the second pass catches any future code paths that construct
`LlmError` variants without going through `build_error_summary`.
