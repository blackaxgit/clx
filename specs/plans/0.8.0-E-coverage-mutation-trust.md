**Goal:** Remove from the coverage denominator the files that are inherently untestable in CI (terminal I/O loops, `unreachable!()` arms, macOS-only keychain code paths, `Debug` impls).

**Files touched:**

- `README.md` (add 1-paragraph honesty note about instrumented denominator)
- `codecov.yml` (NEW; declare the exclusion list publicly)

**Step list:**

1. Run `cargo llvm-cov --workspace --summary-only --json` and capture per-file line%. (~5 min)
2. Identify exclusion targets via this checklist:
   - `crates/clx/src/dashboard/event.rs` (309 LoC, 0% — terminal event loop; W2 reducer will recover most of this, but the residual loop stays out)
   - `crates/clx/src/main.rs` (CLI dispatch; thin wiring covered by `assert_cmd` tests in W6)
   - `crates/clx-hook/src/main.rs` (thin wiring after W3 refactor; the `handle_event` core stays IN)

   - Any file ending in `_debug.rs` or matching `impl Debug for ...` only
3. Add the metadata block to `Cargo.toml`:
   ```toml

   ignore-filename-regex = "(crates/clx/src/dashboard/event\\.rs|crates/clx/src/main\\.rs|crates/clx-hook/src/main\\.rs|crates/clx-core/src/credentials/macos|.*_debug\\.rs)"
   ```
   (~30 min; iterate until `cargo llvm-cov` accepts the regex.)
4. Re-run `cargo llvm-cov --workspace --summary-only` and confirm line% rises into the 80-82% band.
5. Add a 1-paragraph "Coverage denominator" section to `README.md` listing each excluded file plus a 1-line reason. This is the auditability hook.
6. Commit: `chore(ci): exclude untestable files from coverage denominator (W1)`

**Test plan:**
- Manual: re-run `cargo llvm-cov` locally, paste the new % into the PR description for reviewer to verify.
- CI: warn-only gate will pass at the new floor.

**Acceptance:**
- `cargo llvm-cov --workspace --summary-only` reports line% >= 80%.
- `README.md` lists exclusions with reasons.
- No Rust files contain `#[coverage(off)]` (still nightly-only per design §3.6).

**Risks:**
- Over-exclusion looks like coverage theater. Mitigation: each excluded file gets a 1-line rationale in README; reviewer can challenge any entry.
- The regex must match relative paths from workspace root; `cargo llvm-cov` uses unix-style paths even on macOS. Test on both OSes before committing.

## E2. Mutation Testing (`cargo-mutants v27`)

### Goal

Audit assertion quality, not line coverage, on the five security-critical and logic-dense modules where a silent mutation (e.g., flipped `<` to `<=`) would corrupt behavior or leak secrets.

### Hot modules and rationale

| Module | LoC | Why selected | Target kill rate |
|--------|-----|--------------|------------------|
| `crates/clx-core/src/recall.rs` | ~400 | Core ranking logic; off-by-one in RRF or weight handling corrupts retrieval | **80%** |
| `crates/clx-core/src/llm/fallback.rs` | ~250 | Provider fallback correctness; a flipped cooldown comparison reverts to broken-primary forever | **80%** |
| `crates/clx-core/src/storage/migrations.rs` | ~200 | Schema correctness; a mutation that skips a migration could corrupt user DBs silently | **80%** |
| `crates/clx-core/src/policy/mcp.rs` | ~150 | MCP tool allow/deny boundary; already 100% line-covered, mutation audits whether the line coverage is *meaningful* | **80%** |
| `crates/clx-core/src/redaction.rs` | ~180 | Security boundary: a flipped regex could leak secrets in logs/audit; the highest-stakes single file in the codebase | **80%** |

### Configuration files

**`mutants.toml` (NEW, repo root):**

```toml
# cargo-mutants config — hot modules only
examine_globs = [
    "crates/clx-core/src/recall.rs",
    "crates/clx-core/src/llm/fallback.rs",
    "crates/clx-core/src/storage/migrations.rs",
    "crates/clx-core/src/policy/mcp.rs",
    "crates/clx-core/src/redaction.rs",
]
# Skip Debug impls and other low-signal mutations
exclude_re = [
    "impl Debug",
    "impl fmt::Debug",
]
# Skip tests that need network — set a per-test timeout multiplier
test_timeout_multiplier = 2.0
# Default test tool is `cargo test`; matches our CI
```

**`.github/workflows/mutants.yml` (NEW):**

**`.github/workflows/ci.yml` add-on (PR diff job):**

Add a new job `mutants-pr` that runs only if PR diff touches `crates/clx-core/src/(recall|llm/fallback|storage/migrations|policy/mcp|redaction)\.rs`:

```yaml
  mutants-pr:
    name: Mutation testing (PR diff)
    runs-on: macos-latest
    needs: check
    timeout-minutes: 30
    continue-on-error: true   # warn-only for 0.8.0; flip in 0.8.1
    if: github.event_name == 'pull_request'
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
      - name: Check if PR touches hot modules
        id: filter
        run: |
          changed=$(git diff --name-only origin/${{ github.base_ref }}...HEAD)
          if echo "$changed" | grep -qE '^crates/clx-core/src/(recall|llm/fallback|storage/migrations|policy/mcp|redaction)\.rs$'; then
            echo "run=true" >> $GITHUB_OUTPUT
          else
            echo "run=false" >> $GITHUB_OUTPUT
          fi
      - uses: dtolnay/rust-toolchain@stable
        if: steps.filter.outputs.run == 'true'
      - uses: Swatinem/rust-cache@v2
        if: steps.filter.outputs.run == 'true'
      - uses: taiki-e/install-action@v2
        if: steps.filter.outputs.run == 'true'
        with:
          tool: cargo-mutants@27
      - name: Run cargo mutants --check-only on diff
        if: steps.filter.outputs.run == 'true'
        run: cargo mutants --in-place --check-only --no-shuffle
```

### Expected mutation classes and resolution patterns

| Mutation class | Example | Resolution pattern |
|----------------|---------|--------------------|
| Comparison flip (`<` -> `<=`) | RRF `if rank < k` | Add a unit test at the boundary value |
| Arithmetic flip (`+` -> `-`) | Time-decay `age_days / 30` | Property test: monotonic decrease with age |
| Boolean flip (`&&` -> `||`) | `fallback.rs` cooldown gate | Wiremock test with cooldown timing |
| Return-value swap (return `true` always) | `is_non_inert` check | Property test: only listed prefixes match |
| Conditional removal (`if x { ... }` -> ``) | Migration version guard | DB-unit test with fixed `version` state |
| String literal mutation | Regex pattern | Property test: redacted output never contains original |

### Step list

1. **Local dry-run** (~30 min): run `cargo mutants --list` to enumerate the mutant set, sanity-check count is in the 80-200 range.
2. **First full run** (~2 hours wall-clock on M-series): `cargo mutants --in-place > mutants.log`.
3. **Inspect surviving mutants** (~3 hours):
   - For each survivor, classify it (false positive, equivalent mutation, or missing test)
   - For "missing test" survivors: write a unit/property test that kills the mutant
   - For "equivalent" survivors: add to `mutants.toml` `exclude_re` with a 1-line comment justifying
4. **Iterate** until kill rate >= 80% on each module.
5. Add `mutants.toml` to repo.
6. Add `.github/workflows/mutants.yml` (weekly) and the `mutants-pr` job to `ci.yml`.
7. Commit:
   - `chore(ci): add mutants.toml configuration`
   - `test(recall): kill mutants in RRF boundary` (per module)
   - `test(fallback): kill cooldown comparison mutants`
   - `test(migrations): kill version guard mutants`
   - `test(policy-mcp): kill allow/deny mutants`
   - `test(redaction): kill regex pattern mutants`
   - `ci(mutants): weekly baseline + PR diff jobs`

### Test plan

- Local: `cargo mutants --in-place` on each module.
- CI: PR diff job (warn-only in 0.8.0), weekly full job.

### Acceptance

- `cargo mutants --in-place` on each hot module: kill rate >= 80%.
- Weekly workflow runs green on `main`.
- PR diff job triggers when expected (verify with a deliberate one-line change in `redaction.rs`).

### Risks

| Risk | Mitigation |
|------|------------|
| Full run wall-clock blows past 30 min on macOS GHA | Use `--shard 0/4` etc. across multiple jobs; or move to ubuntu-latest for the diff job (mac only for the weekly) |
| Mutations on `tracing::warn!` macros are noise | Add `exclude_re = ["tracing::"]` to `mutants.toml` |
| Flaky test gives false mutant survival | Mutation test inherits flakiness; mitigate by requiring full `cargo test --workspace` green before any mutation run |
| Survivor count too high to triage in 0.8.0 | If >40 survivors, defer non-security modules to 0.8.1 and keep `redaction.rs` + `policy/mcp.rs` as the must-hit-80% pair |

## E3. `clx trust` Command

### Goal

Allow power users to opt-in to non-inert project config keys (e.g., `providers.azure.endpoint`) by trusting a specific file hash. Trust is per-machine, per-user, per-file-hash. It does NOT propagate via git.

### File layout

```
crates/clx-core/src/config/
  mod.rs                  # re-export trust module
  project.rs              # MODIFIED: insert trust check at line 86
  trust.rs                # NEW (~150 LoC)

crates/clx/src/commands/
  trust.rs                # NEW (~120 LoC)
  mod.rs                  # MODIFIED: register `trust` subcommand

crates/clx/src/main.rs    # MODIFIED: route Cli::Trust
```

### Storage format

**Path:** `<config_dir>/trusted_configs.json` (where `<config_dir>` is `~/.clx/` on Unix, derived from `clx-core::paths`).

**Schema:**

```json
{
  "version": 1,
  "entries": [
    {
      "hash": "sha256:abc123...64hexchars",
      "path": "/Users/x/repo/.clx/config.yaml",
      "added_at": "2026-05-16T10:00:00Z",
      "added_by": "clx-trust-cli@0.8.0"
    }
  ]
}
```

Permissions: `0600` on Unix (chmod after write).

### Module: `crates/clx-core/src/config/trust.rs`

**Public surface:**

```rust
//! Trusted-config registry. Per-machine, per-user, per-file-hash.
//! Not propagated via git.

use std::path::Path;

pub struct TrustEntry {
    pub hash: String,           // "sha256:..." prefix included
    pub path: std::path::PathBuf,
```

```rust
#[derive(Subcommand)]
pub enum TrustCommand {
    /// Add the current project's config to the trustlist.
    Add { path: Option<std::path::PathBuf> },
    /// List trusted configs.
    List {
        #[arg(long)]
        json: bool,
    },
    /// Remove by hash prefix.
    Remove { hash: String },
}

pub fn run(cmd: TrustCommand) -> anyhow::Result<()> {
    let config_dir = clx_core::paths::config_dir()?;
    let mut tl = clx_core::config::trust::TrustList::load(&config_dir)?;
    match cmd {
        TrustCommand::Add { path } => add(&mut tl, path, &config_dir),
        TrustCommand::List { json } => list(&tl, json),
        TrustCommand::Remove { hash } => remove(&mut tl, &hash, &config_dir),
    }
}
```
