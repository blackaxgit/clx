## Item 1 ‚Äî R-B1-4/B3-2 + Net-New Bash(*)x: `parse_pattern` trailing-junk bypass

### Location

`crates/clx-core/src/policy/matching.rs` lines 10‚Äì26 (pre-fix)

### Root Cause

`parse_pattern` used `rfind(')')` to find the closing parenthesis but did **not**
verify the `)` was the last non-whitespace character. For input `"Bash(*)x"`:

- `find('(')` ‚Üí 4  
- `rfind(')')` ‚Üí 6 (the `)` inside, before the trailing `x`)  
- `paren_end (6) > paren_start (4)` ‚Üí passes the structural check  
- Returns `Some(("Bash", "*"))`

`PolicyEngine::matches_rule` then calls `glob_match("*", command)` which matches
every Bash command, producing `PolicyDecision::Allow` for arbitrary commands
including `rm -rf /`.

The `is_overbroad_allow_pattern("Bash(*)x")` check returns `false` (the trailing
`x` makes it non-overbroad from the detector's perspective), so this variant also
evades the per-pattern gate applied on the learned-rule and MCP-add paths.

### Fix

`crates/clx-core/src/policy/matching.rs` ‚Äî after finding `paren_end` via
`rfind`, check that the substring after the `)` (`pattern[paren_end+1..]`) is
empty or whitespace-only. If not, return `None`.

```rust
// Security: reject patterns where non-whitespace characters follow the
// closing `)`.  Trailing whitespace is harmless; trailing text is not.
let after_paren = &pattern[paren_end + 1..];
if !after_paren.trim().is_empty() {
    return None;
}
```

## Item 2 ‚Äî R-B1-4/B3-2 / T3: File-loaded whitelist bypasses overbroad gate

### Location

`crates/clx-core/src/policy/rules.rs` lines 216‚Äì219 (pre-fix)

### Root Cause

`load_rules_from_file` pushed every whitelist pattern from a YAML config file
directly into `self.whitelist` with no overbroad check:

```rust
for pattern in config.whitelist {
    self.whitelist
        .push(PolicyRule::whitelist(pattern).with_source(RuleSource::Config));
}
```

A file containing `whitelist: ["Bash(*)"]` would insert an unrestricted wildcard
allow rule into L0. Subsequent `evaluate("Bash", <anything>)` calls returned
`PolicyDecision::Allow` regardless of command content. The overbroad gate added
by Stream G3 of the prior round only applied to the `load_learned_rules` path
(database-loaded rules); the file-load path was not gated.

### Fix

`crates/clx-core/src/policy/rules.rs` ‚Äî apply `is_overbroad_allow_pattern`
before pushing each whitelist entry. WARN and skip (do not abort); deny rules
are unaffected.

```rust
for pattern in config.whitelist {
    // B1-4 / B3-2: apply the same overbroad gate that guards the
    // learned-rule load path.
    if super::matching::is_overbroad_allow_pattern(&pattern) {
        warn!(
            %pattern,
            "Skipping overbroad file-loaded ALLOW rule (would whitelist \
             arbitrary commands); ignored at load"
        );
        continue;
    }
    self.whitelist
        .push(PolicyRule::whitelist(pattern).with_source(RuleSource::Config));
}
```
